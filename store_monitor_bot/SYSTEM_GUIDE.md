ماذا يحدث خطوة بخطوة عندما يرسل المستخدم رسالة؟
──────────────────────────────────────────────────

1. المستخدم يرسل رسالة في تيليغرام (مثلاً يضغط "➕ إضافة منتج").

2. تيليغرام ترسل الرسالة إلى البوت عبر Long Polling.
   (الملف: main.py → dp.start_polling)

3. أول شيء يحدث: فحص الحماية من الإرسال المتكرر (ThrottleMiddleware).
   - إذا أرسل المستخدم أكثر من 5 رسائل خلال 30 ثانية → تُرفض الرسالة بصمت.
   - إذا كان طبيعياً → تمر الرسالة.
   (الملف: bot/middleware/throttle.py)

4. ثم يفتح النظام جلسة مع قاعدة البيانات (DatabaseMiddleware).
   يُمرر كائن session و connector_manager للـ handler.
   (الملف: main.py → DatabaseMiddleware)

5. البوت يبحث عن الـ handler المناسب:
   - النص "➕ إضافة منتج" يُطابق handler اسمه add_product_start
   - البوت يُنفذ الدالة المطابقة
   (الملف: bot/handlers/user.py)

6. الـ handler يتعامل مع قاعدة البيانات (يقرأ/يكتب) ويرد على المستخدم.

7. بعد انتهاء الـ handler: الجلسة تُغلق تلقائياً.


الملفات الرئيسية ووظائفها:
─────────────────────────

  main.py                    → نقطة البداية: يشغّل كل شيء معاً
  config/settings.py         → جميع الإعدادات (توكن، حدود الخطط، أوزان التقييم)
  db/models.py               → تعريف جداول قاعدة البيانات (11 جدول)
  db/crud.py                 → دوال القراءة والكتابة من/إلى قاعدة البيانات
  core/monitor.py            → محرك المراقبة (يعمل في الخلفية)
  core/connectors/amazon.py  → موصل أمازون
  core/connectors/generic.py → موصلات Shopify و WooCommerce و Generic + المدير
  bot/handlers/user.py       → معالجات المستخدم (إضافة منتج، قائمة المنتجات...)
  bot/handlers/user2.py      → معالجات إضافية (عروض، تقارير، اشتراكات، إعدادات...)
  bot/handlers/admin.py      → معالجات الإدارة (اعتماد فرص، إعلانات...)
  bot/keyboards/main.py      → جميع أزرار البوت
  admin/dashboard.py         → لوحة الإدارة على الويب (FastAPI)
  auth/security.py           → نظام الحماية (JWT، CSRF، تحديد معدل الطلبات)
  middleware/security.py      → حماية لوحة الإدارة (CORS، Headers، حد الحجم)
  utils/url_validator.py     → حماية SSRF (التحقق من الروابط قبل السكرابينج)

──────────────────────────────────────
1. ➕ إضافة منتج
──────────────────────────────────────

ماذا يحدث عند الضغط:
  البوت يطلب منك إرسال رابط المنتج.
  الرسالة: "🔗 أرسل رابط المنتج الذي تريد مراقبته"

ما يتوقعه البوت من المستخدم:
  رابط يبدأ بـ https:// يؤدي إلى صفحة منتج.
  مثال صحيح: https://www.amazon.sa/dp/B08L5TNJHG
  مثال خاطئ: www.amazon.sa/dp/B08L5TNJHG (بدون https://)
  مثال خاطئ: amazon.sa (صفحة رئيسية وليست منتج)

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user.py → دالة process_product_url
  2. يتحقق: هل الرابط يبدأ بـ http:// أو https://؟
     → لا؟ رسالة خطأ: "❌ الرابط غير صالح"
  3. يتحقق: هل المستخدم وصل للحد الأقصى لخطته؟
     → مجاني = 3 منتجات، أساسي = 50، احترافي = 300
     → وصل الحد؟ رسالة: "❌ وصلت للحد الأقصى!"
  4. يرسل الرابط لمدير الموصلات (ConnectorManager)
     → الملف core/connectors/generic.py → دالة scrape
     → أولاً يتحقق من سلامة الرابط ضد SSRF (حماية أمنية)
     → ثم يجرب الموصلات بالترتيب: أمازون → شوبيفاي → ووكومرس → عام
  5. الموصل المناسب يفتح الصفحة ويستخرج: الاسم، السعر، المخزون، التقييم...
  6. تظهر لك بيانات المنتج مع أزرار:

  الأزرار التي تظهر بعد العثور على المنتج:
  ┌────────────────────────────────┐
  │  ✅ بدء المراقبة               │  → يحفظ المنتج ويبدأ مراقبته
  │  🔔 إعداد التنبيهات            │  → لتخصيص نوع التنبيهات
  │  ❌ إلغاء    │  🏠 رجوع       │  → يلغي العملية
  └────────────────────────────────┘

ماذا يرى المستخدم في النهاية:
  "✅ تمت الإضافة بنجاح!
   📦 اسم المنتج
   سيتم إشعارك عند تغير السعر أو المخزون."

أخطاء شائعة وكيفية تجنبها:
  ✖ إرسال رابط بدون https → أضف https:// في البداية
  ✖ إرسال رابط صفحة رئيسية بدلاً من منتج → تأكد أنك في صفحة المنتج نفسه
  ✖ إرسال رابط متجر غير مدعوم → استخدم "طلب إضافة متجر"
  ✖ الوصول لحد الخطة → احذف منتجات قديمة أو رقّ اشتراكك

مثال عملي كامل مع رابط أمازون:
  1. اضغط "➕ إضافة منتج"
  2. انسخ رابط المنتج من أمازون:
     https://www.amazon.sa/dp/B0BSHF7WHJ
  3. ألصقه في المحادثة واضغط إرسال
  4. البوت يرد: "⏳ جاري فحص المنتج..."
  5. بعد 2-5 ثوانٍ يظهر:
     "✅ تم التعرف على المنتج!
      📦 Apple AirPods Pro (2nd generation)
      💰 السعر الحالي: 899.00 SAR
      🏪 المتجر: Amazon
      📦 المخزون: متوفر ✅
      ⭐ التقييم: 4.7/5"
  6. اضغط "✅ بدء المراقبة"
  7. تظهر: "✅ تمت الإضافة بنجاح!"
  → من الآن سيُنبهك عند أي انخفاض في السعر.


──────────────────────────────────────
2. 📦 منتجاتي
──────────────────────────────────────

ماذا يحدث عند الضغط:
  يعرض قائمة بجميع المنتجات التي تراقبها.
  كل منتج يظهر كزر مستقل فيه: اسمه + سعره + حالته (▶️ نشط / ⏸ موقوف).

ما يتوقعه البوت:
  لا شيء — فقط يعرض القائمة.

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user.py → دالة my_products
  2. يجلب المستخدم من الـ DB بواسطة telegram_id
  3. يجلب منتجاته من جدول user_products (مع selectinload للمنتج)
  4. يعرض القائمة

ماذا يرى المستخدم:
  إذا لا توجد منتجات:
    "📦 منتجاتك المراقبة: لا توجد منتجات بعد!"

  إذا توجد منتجات:
    "📦 منتجاتك المراقبة (3 منتج)
     1. ▶️ Apple AirPods Pro   💰 899 | ✅
     2. ▶️ Samsung Galaxy S24  💰 3499 | ✅
     3. ⏸ Sony WH-1000XM5    💰 1299 | ❌"

  مع أزرار لكل منتج + أزرار إضافية:
  ┌──────────────────────────────────────┐
  │  ▶️ Apple AirPods Pro - 899          │  → يفتح تفاصيل المنتج
  │  ▶️ Samsung Galaxy S24 - 3499        │  → يفتح تفاصيل المنتج
  │  ⏸ Sony WH-1000XM5 - 1299           │  → يفتح تفاصيل المنتج
  │  ➕ إضافة منتج                       │  → يبدأ عملية إضافة جديدة
  │  🔄 تحديث الكل                       │  → يحدث البيانات
  │  🔀 فرز المنتجات                     │  → فرز حسب السعر/التاريخ/الاسم
  │  🏠 رجوع للرئيسية                    │  → يرجع للقائمة الرئيسية
  └──────────────────────────────────────┘

عند الضغط على منتج → صفحة التفاصيل:
  "📦 Apple AirPods Pro
   💰 السعر الحالي: 899.00 SAR
   📉 أقل سعر: 849.00 SAR
   📈 أعلى سعر: 1099.00 SAR
   🏪 المخزون: متوفر ✅
   ⏰ آخر تحديث: قبل 15 دقيقة"

  أزرار صفحة التفاصيل:
  ┌────────────────────────┬────────────────────────┐
  │  🔄 تحديث الآن         │  🔔 إعداد التنبيهات    │
  │  📈 سجل السعر          │  📦 سجل المخزون        │
  │  ⏸ إيقاف المراقبة      │  🗑 حذف المنتج         │
  │         🔙 رجوع                                 │
  └─────────────────────────────────────────────────┘

  شرح كل زر فرعي:

  🔄 تحديث الآن → يرسل طلب سكرابينج فوري لهذا المنتج
  🔔 إعداد التنبيهات → يعرض 6 أنواع تنبيهات (مشروحة في الجزء 3)
  📈 سجل السعر → يعرض آخر 10 تغييرات في السعر مع التاريخ
  📦 سجل المخزون → يعرض تاريخ تغيرات حالة التوفر
  ⏸ إيقاف المراقبة / ▶️ استئناف المراقبة → يوقف/يستأنف التنبيهات لهذا المنتج
  🗑 حذف المنتج → يحذف نهائياً (يطلب تأكيد أولاً):
    "⚠️ هل أنت متأكد من حذف هذا المنتج؟"
    [✅ نعم, احذف] [❌ لا, إلغاء]


──────────────────────────────────────
3. 📂 مراقبة فئة
──────────────────────────────────────

ماذا يحدث عند الضغط:
  البوت يطلب منك رابط فئة (قسم) في متجر.
  مثال: قسم الهواتف في أمازون.

ما يتوقعه البوت:
  رابط يبدأ بـ http:// أو https://
  مثال: https://www.amazon.sa/s?rh=n:12345

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user2.py → دالة monitor_category_start
  2. يدخل حالة FSM: AddCategoryStates.waiting_for_url
  3. عند إرسال الرابط → دالة process_category_url
  4. يتحقق: هل خطة المستخدم تسمح بمراقبة الفئات؟
     → الخطة المجانية: max_categories = 0 → ممنوعة!
     → الأساسية: max_categories = 10
     → الاحترافية: max_categories = 50
  5. إذا مسموح → يحفظ الفئة في جدول user_categories

ماذا يرى المستخدم:
  إذا مجاني:
    "❌ مراقبة الفئات غير متاحة في الخطة المجانية.
     قم بترقية اشتراكك للوصول لهذه الميزة."

  إذا مسموح:
    "✅ تم إضافة الفئة للمراقبة!
     🔗 https://www.amazon.sa/s?rh=n:12..."

  أزرار بعد إضافة الفئة:
  ┌────────────────────────────────┐
  │  🔔 إعداد تنبيهات الفئة       │
  │  👁 عرض الفئة                  │
  │  🗑 حذف الفئة                  │
  │  🏠 رجوع للرئيسية              │
  └────────────────────────────────┘

⚠️ ملاحظة مهمة: ميزة مراقبة الفئات حالياً تحفظ الفئة في قاعدة البيانات
   لكن محرك المراقبة لا يفحص الفئات تلقائياً بعد (ميزة قيد التطوير).


──────────────────────────────────────
4. 🏪 مراقبة متجر
──────────────────────────────────────

ماذا يحدث عند الضغط:
  البوت يتحقق من خطتك أولاً.
  هذه الميزة متاحة للاشتراك الاحترافي فقط (49 ريال).

ما يتوقعه البوت:
  رابط المتجر الرئيسي.
  مثال: https://www.extra.com

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user2.py → دالة monitor_store_start
  2. يتحقق: PLAN_LIMITS[user.plan]["max_stores"]
     → المجانية: 0 → ممنوعة
     → الأساسية: 0 → ممنوعة
     → الاحترافية: 20 → مسموحة

ماذا يرى المستخدم:
  إذا لم يكن احترافي:
    "❌ مراقبة المتاجر متاحة في الاشتراك الاحترافي فقط.
     قم بالترقية للوصول لهذه الميزة."

  إذا احترافي:
    يطلب رابط المتجر → يحفظه.

⚠️ ملاحظة: مثل مراقبة الفئات، الحفظ يعمل لكن الفحص التلقائي قيد التطوير.


──────────────────────────────────────
5. 🔥 أفضل العروض
──────────────────────────────────────

ماذا يحدث عند الضغط:
  يعرض أفضل 10 عروض اعتمدتها الإدارة، مرتبة حسب نسبة الخصم.

ما يتوقعه البوت:
  لا شيء — فقط يعرض.

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user2.py → دالة best_deals
  2. يبحث في جدول opportunities عن الفرص بحالة APPROVED
  3. يرتبها حسب discount_percent تنازلياً
  4. يأخذ أول 10

ماذا يرى المستخدم:
  إذا لا توجد عروض:
    "🔥 أفضل العروض: لا توجد عروض متاحة حالياً."

  إذا توجد:
    "🔥 أفضل عروض اليوم (5 عرض)
     1. Apple AirPods Pro
        خصم 25% | السعر: 674
     2. Samsung Galaxy Watch
        خصم 20% | السعر: 799
     ..."

  مع أزرار لكل عرض:
  ┌────────────────────────────────────────┐
  │  🔥 Apple AirPods Pro - خصم 25%        │
  │  🔥 Samsung Galaxy Watch - خصم 20%     │
  │  ...                                   │
  │  🔄 تحديث                              │
  │  🏠 رجوع للرئيسية                      │
  └────────────────────────────────────────┘

  عند الضغط على عرض → صفحة تفاصيل العرض:
    "🔥 عرض قوي!
     📦 المنتج: Apple AirPods Pro
     💰 السعر السابق: 899.00
     🔥 السعر الحالي: 674.00
     📉 الخصم: 25.0%
     📦 المخزون: متوفر ✅"

    أزرار:
    ┌──────────────────────┬──────────────────────┐
    │  🛒 شراء الآن        │  👁 راقب المنتج       │
    │         🔙 رجوع                              │
    └──────────────────────────────────────────────┘

    🛒 شراء الآن → يفتح رابط الأفلييت (أو رابط المنتج الأصلي)
    👁 راقب المنتج → يضيف المنتج لقائمة مراقبتك مباشرة
      → إذا وصلت لحد خطتك → "❌ وصلت للحد الأقصى! قم بترقية الاشتراك."


──────────────────────────────────────
6. 📊 التقارير
──────────────────────────────────────

ماذا يحدث عند الضغط:
  يعرض ملخص إحصائي لنشاطك اليوم.

ما يتوقعه البوت:
  لا شيء.

ماذا يحدث خلف الكواليس:
  1. الملف bot/handlers/user2.py → دالة reports
  2. يجلب المنتجات النشطة للمستخدم (ACTIVE)
  3. يحسب عدد تغيرات الأسعار اليوم (من جدول price_history)
  4. يحسب عدد العروض القوية اليوم (من جدول opportunities)

ماذا يرى المستخدم:
  "📊 تقرير اليوم
   🔄 5 تغيير في الأسعار
   🔥 2 عروض قوية
   📦 3 منتج تحت المراقبة"

  أزرار:
  ┌──────────────┬──────────────┬──────────────┐
  │ 📅 تقرير    │ 📅 تقرير    │ 📅 تقرير     │
  │    يومي     │    أسبوعي   │    شهري      │
  ├──────────────┴──────────────┴──────────────┤
  │          🏠 رجوع للرئيسية                   │
  └─────────────────────────────────────────────┘

⚠️ ملاحظة: أزرار التقرير اليومي/الأسبوعي/الشهري موجودة في الواجهة
   لكن callbacks لم تُربط بعد (ميزة قيد التطوير).


──────────────────────────────────────
7. 💳 الاشتراك
──────────────────────────────────────

ماذا يحدث عند الضغط:
  يعرض خطتك الحالية وحدودها مع خيارات للترقية.

ماذا يرى المستخدم:
  "💳 خطتك الحالية: 🆓 مجانية
   📦 المنتجات: 3
   📂 الفئات: غير متاح
   🏪 المتاجر: غير متاح
   ⏱ فحص كل: 60 دقيقة"

  أزرار:
  ┌──────────────────────────────────────────┐

الجزء 5: سير عمل الإدارة — خطوة بخطوة
════════════════════════════════════════


──────────────────────────────────────
الخطوة 1: كيف يكتشف النظام الفرصة؟
──────────────────────────────────────

المحرك يفحص المنتجات كل 60 ثانية. عند اكتشاف انخفاض في السعر:

1. يحسب نسبة الخصم:
   نسبة الخصم = ((السعر القديم - السعر الجديد) / السعر القديم) × 100

2. يتحقق: هل نسبة الخصم ≥ MIN_DISCOUNT_PERCENT (= 10%)؟
   → أقل من 10%؟ لا تعتبر فرصة.
   → 10% أو أكثر؟ تعتبر فرصة → يحسب النقاط.

3. حساب نقاط الفرصة (OpportunityScorer) — المجموع من 100:

   ┌────────────────────────┬────────┬──────────────────────────────────────┐
   │ العامل                 │ الوزن  │ كيف يُحسب                            │
   ├────────────────────────┼────────┼──────────────────────────────────────┤
   │ نسبة الخصم             │ 40 نقطة│ خصم% × 0.8 (بحد أقصى 40)            │
   │ تقييم المنتج           │ 20 نقطة│ (التقييم/5) × 20                     │
   │ عدد المراجعات          │ 15 نقطة│ (العدد/1000) × 15 (بحد أقصى 15)     │
   │ توافر المخزون          │ 10 نقاط│ متوفر = 10، غير متوفر = 0           │
   │ أقل سعر تاريخي؟       │ 15 نقطة│ أقل من أي وقت = 15، قريب = 7       │
   └────────────────────────┴────────┴──────────────────────────────────────┘

4. تصنيف الفرصة:
   90+ نقطة = 🔥 ممتاز
   70-89 نقطة = ✅ جيد
   أقل من 70 = ℹ️ عادي

مثال بالأرقام:
  منتج: Samsung Galaxy S24
  السعر القديم: 3499 ريال
  السعر الجديد: 2449 ريال
  نسبة الخصم: ((3499-2449)/3499) × 100 = 30%
  
  حساب النقاط:
    خصم (30% × 0.8 = 24)                          = 24 نقطة
    تقييم (4.5/5 × 20 = 18)                        = 18 نقطة  
    مراجعات (2500/1000 × 15 = 37.5 → بحد أقصى 15) = 15 نقطة
    مخزون (متوفر)                                   = 10 نقاط
    أقل سعر تاريخي؟ (نعم)                           = 15 نقطة
    ─────────────────────────────────────────────
    المجموع                                         = 82 نقطة → ✅ جيد


──────────────────────────────────────
الخطوة 2: ماذا يستقبل المدير في تيليغرام؟
──────────────────────────────────────

تصل رسالة لمجموعة الإدارة (ADMIN_GROUP_ID):

  "💡 فرصة جديدة

   📦 المنتج: Samsung Galaxy S24
   🏪 المتجر: https://www.amazon.sa/dp/B0...
   💰 السعر السابق: 3499.00
   🔥 السعر الحالي: 2449.00
   📉 الخصم: 30.0%
   📦 المخزون: متوفر ✅
   ⭐ التقييم: ✅ جيد (82/100)
   🔗 https://www.amazon.sa/dp/B0XXXXX..."

  أزرار:
  ┌──────────────────────┬──────────────────────┐
  │  ✅ اعتماد وإرسال    │  ✏️ تعديل الرسالة    │
  │  ⏰ تأجيل            │  ❌ تجاهل             │
  └──────────────────────┴──────────────────────┘

  شرح كل زر:

  ✅ اعتماد وإرسال:
    → يعتمد الفرصة فوراً
    → يُغير حالتها إلى APPROVED في قاعدة البيانات
    → يُسجل published_at بالوقت الحالي
    → يرسل العرض لجميع المستخدمين الذين يراقبون هذا المنتج

  ✏️ تعديل الرسالة:
    → يسمح للمدير بكتابة نص مخصص يُضاف للعرض

  ⏰ تأجيل:
    → يغير الحالة إلى POSTPONED
    → الفرصة تبقى ويمكن مراجعتها لاحقاً

  ❌ تجاهل:
    → يغير الحالة إلى REJECTED
    → الفرصة تختفي من القائمة


──────────────────────────────────────
الخطوة 3: كيف تضيف رابط الأفلييت
──────────────────────────────────────

عند فتح تفاصيل الفرصة من "💡 مراجعة الفرص":

1. اضغط "🔗 إضافة رابط أفلييت"
2. البوت يطلب: "أرسل رابط الأفلييت الخاص بهذا المنتج"
3. اذهب إلى صفحة المنتج على أمازون
4. استخدم برنامج Amazon Associates لإنشاء رابط عمولة
   أو أضف tag يدوياً: ?tag=yourstore-21
5. ألصق الرابط وأرسله
6. البوت يؤكد:
   "✅ تم حفظ الرابط. هل تريد إرسال العرض الآن؟"
   [✅ إرسال الآن بدون تعديل] [✏️ تعديل النص أولاً] [❌ إلغاء]
7. اضغط "إرسال الآن" → يُرسل العرض مع رابط الأفلييت

كيف يظهر الرابط في رسالة العرض:
  زر "🛒 شراء الآن" في رسالة العروض يستخدم رابط الأفلييت.
  إذا لم تضف أفلييت → يستخدم الرابط الأصلي للمنتج.


──────────────────────────────────────
الخطوة 4: كيف يصل العرض للمشتركين؟
──────────────────────────────────────

من يستقبل العرض:
  ✅ فقط المستخدمون الذين يراقبون هذا المنتج بالذات
  ✅ وحالة مراقبتهم ACTIVE (ليست PAUSED أو DELETED)
  ❌ المستخدمون الذين كتموا التنبيهات (muted = True) لا يستقبلون
  ❌ المستخدمون المحظورون (is_banned = True) لا يستقبلون

الرسالة التي يستقبلها المشترك:
  "🔥 عرض قوي!

   📦 Samsung Galaxy S24
   💰 السعر السابق: 3499.00
   🔥 السعر الحالي: 2449.00
   📉 خصم 30.0%
   📦 المخزون: متوفر ✅

   💬 عرض محدود! (إذا أضاف المدير نصاً مخصصاً)

   🛒 اشترِ الآن"  ← رابط يؤدي للأفلييت أو المنتج

الكود المسؤول:
  الملف bot/handlers/admin.py → دالة send_deal_to_subscribers
  يرسل الرسائل واحدة تلو الأخرى لكل مستخدم.


──────────────────────────────────────
الخطوة 5: لوحة الإدارة على الويب (Dashboard)
──────────────────────────────────────

كيف تفتحها:
  1. تأكد أن main.py يعمل
  2. افتح المتصفح على: http://localhost:8000
  3. تظهر صفحة تسجيل دخول
  4. أدخل Telegram ID الخاص بك (يجب أن يكون في ADMIN_USER_IDS)
  5. اضغط "دخول"
  6. تحصل على JWT token يُخزن تلقائياً

أقسام اللوحة:

  📊 الرئيسية → إحصائيات سريعة:
    - عدد المشتركين
    - عدد المنتجات المراقبة
    - عدد الفرص الجديدة
    - عدد العروض المرسلة اليوم
    + جدول بأحدث الفرص

  💡 الفرص → قائمة كاملة بالفرص مع فلترة حسب الحالة:
    - جديدة / معتمدة / مرفوضة
    - أزرار اعتماد ✅ ورفض ❌ لكل فرصة

  👥 المستخدمون → قائمة المستخدمين مع:
    - الاسم، الخطة، عدد المنتجات، تاريخ التسجيل
    - فلترة حسب الخطة

  🏪 المتاجر → قائمة المتاجر المدعومة

  💊 صحة النظام → حالة:
    - قاعدة البيانات: ok/error
    - Redis: ok/error
    - محرك المراقبة: ok/stopped
    - آخر وقت فحص

الفرق بين الداشبورد وتيليغرام:
  الداشبورد يعرض البيانات بشكل أشمل (جداول، فلاتر، إحصائيات).
  تيليغرام أسرع للعمليات اليومية (اعتمد/ارفض بضغطة واحدة).
  كلاهما يصل لنفس قاعدة البيانات.


════════════════════════════════════════
الجزء 6: محرك المراقبة — كيف يعمل
════════════════════════════════════════

بلغة بسيطة:
"كل 60 ثانية يستيقظ النظام ويقوم بالتالي..."

الخطوة 1: الاستيقاظ
  محرك المراقبة (MonitoringEngine) يعمل في حلقة لا نهائية.
  ينام 60 ثانية → يستيقظ → يبدأ دورة فحص → ينام 60 ثانية → وهكذا.

الخطوة 2: جلب المنتجات المطلوبة
  يسأل قاعدة البيانات: "أعطني المنتجات التي لم تُفحص منذ 30 دقيقة"
  يأخذ أقصى 50 منتج لكل دورة (لمنع الضغط على المتاجر).
  يختار فقط المنتجات التي لها مراقبون نشطون (status = ACTIVE).

الخطوة 3: الفحص المتزامن
  يفحص المنتجات بالتوازي (حتى 10 منتجات في نفس الوقت).
  بين كل مجموعة وأخرى ينتظر SCRAPING_DELAY_SECONDS (= 2 ثانية).
  هذا يمنع حظر البوت من المتاجر.

الخطوة 4: لكل منتج
  1. يُرسل الرابط لمدير الموصلات
  2. الموصل يزور صفحة المنتج ويستخرج البيانات
  3. يقارن السعر الجديد بالقديم
  4. يُحدّث قاعدة البيانات:
     - current_price = السعر الجديد
     - يضيف سجل في price_history
     - يحدث lowest_price / highest_price
     - last_scraped = الآن

الخطوة 5: اكتشاف الفرص
  إذا السعر انخفض ≥ 10% → يحسب نقاط الفرصة → يُنشئ Opportunity
  → يُرسل إشعار لمجموعة الإدارة

الخطوة 6: إرسال التنبيهات
  يمر على جميع المستخدمين الذين يراقبون هذا المنتج:
  - يتحقق من نوع التنبيه المفعّل لكل مستخدم
  - يطابق الشروط (هل انخفض السعر؟ هل نفد المخزون؟...)
  - يرسل التنبيه المناسب


لماذا 50 منتج فقط لكل دورة:
  إذا فحص كل المنتجات مرة واحدة (مثلاً 5000 منتج) يحدث:
  1. ضغط هائل على المتاجر → قد يُحظر البوت
  2. بطء شديد في الدورة الواحدة
  3. ضغط على قاعدة البيانات
  الحل: 50 منتج × دورة كل 60 ثانية = 3000 منتج/ساعة (كافٍ لمعظم الحالات)


كيف يوفر فحص الرابط الواحد مرة واحدة:
  مثال: 100 مستخدم يراقبون نفس المنتج على أمازون.
  بدون تحسين: 100 طلب لنفس الصفحة!
  مع التحسين: النظام يختار DISTINCT products = فحص واحد فقط.
  ثم يُرسل النتائج لجميع الـ 100 مستخدم.


ماذا يحدث عند فشل السكرابينج:
  1. المحاولة الأولى تفشل → ينتظر 5 ثوانٍ
  2. المحاولة الثانية تفشل → ينتظر 10 ثوانٍ
  3. المحاولة الثالثة تفشل → يتخطى المنتج (يُحاول في الدورة التالية)
  (هذا يسمى Exponential Backoff)


كيف يتجنب النظام الحظر:
  1. تأخير 2 ثانية بين كل فحص وآخر
  2. بحد أقصى 10 طلبات في الدقيقة لكل موقع
  3. إعادة محاولة بانتظار متزايد عند الفشل
  4. User-Agent يحاكي متصفحاً حقيقياً (Chrome)
  5. لأمازون: في حالة فشل HTTP → يستخدم Playwright (متصفح كامل)

الجزء 7: دليل الموصلات (Scrapers)

──────────────────────────────────────
🛒 أمازون (Amazon)
──────────────────────────────────────

الدومينات المدعومة:
  amazon.com, amazon.sa, amazon.ae, amazon.co.uk,
  amazon.de, amazon.fr, amazon.com.au, amazon.ca

صيغ الروابط التي تعمل:
  ✅ https://www.amazon.sa/dp/B08L5TNJHG
  ✅ https://www.amazon.com/dp/B08L5TNJHG
  ✅ https://www.amazon.sa/product/B08L5TNJHG
  ✅ https://www.amazon.sa/some-product-name/dp/B08L5TNJHG/ref=...

صيغ لا تعمل:
  ❌ https://www.amazon.sa/ (الصفحة الرئيسية)
  ❌ https://www.amazon.sa/s?k=headphones (صفحة بحث)
  ❌ amazon.sa/dp/B08L5TNJHG (بدون https://)

كيف تحصل على أنظف رابط:
  1. افتح صفحة المنتج في أمازون
  2. من شريط العنوان، ابحث عن /dp/XXXXXXXXXX
  3. انسخ: https://www.amazon.sa/dp/XXXXXXXXXX فقط
  4. احذف كل شيء بعد الـ ASIN (10 أحرف بعد /dp/)

مثال:
  رابط طويل: https://www.amazon.sa/Apple-Generation-Cancelling-Transparency-Personalized/dp/B0BDHWDR12/ref=sr_1_1?crid=2X5C&keywords=airpods&qid=1679&s=electronics&sprefix=air%2Celectronics%2C259&sr=1-1
  رابط نظيف: https://www.amazon.sa/dp/B0BDHWDR12
  كلاهما يعمل! لكن النظيف أفضل.

البيانات المستخرجة:
  الاسم، السعر، العملة، المخزون، التقييم، عدد المراجعات، الصورة، ASIN

طريقة الاستخراج:
  أولاً يحاول HTTP عادي (أسرع: 2-5 ثوانٍ)
  إذا فشل → يستخدم Playwright متصفح كامل (أبطأ: 10-15 ثانية)

صيغة الرابط:
  ✅ https://store.com/product/product-name

البيانات المستخرجة:

──────────────────────────────────────
🌐 المتاجر العامة (Generic)
──────────────────────────────────────

متى يعمل الموصل العام:
  عندما لا يتعرف النظام على المتجر كأمازون أو شوبيفاي أو ووكومرس.
  يحاول استخدام:
  1. JSON-LD (بيانات هيكلية في HTML)
  2. Open Graph meta tags (og:title, og:price)

ما يمكن استخراجه:
  الاسم، السعر (إذا موجود في البيانات الهيكلية)

ما لا يمكن استخراجه:
  المخزون، التقييم، المراجعات (يعتمد على الموقع)

متى تستخدم "طلب إضافة متجر":
  إذا جربت رابط من متجر والبوت قال "لم نتمكن من جلب البيانات"
  → هذا يعني أن الموصل العام فشل أيضاً
  → استخدم "🏬 طلب إضافة متجر" لإعلام الإدارة

الجزء 8: دليل حل المشاكل
════════════════════════════════════════


──────────────────────────────────────
❌ "لم نتمكن من جلب بيانات المنتج"
──────────────────────────────────────

السبب 1: الرابط لا يؤدي لصفحة منتج
  → تأكد أن الرابط يفتح صفحة منتج محدد (ليس صفحة بحث أو صفحة رئيسية)
  → الحل: افتح الرابط في المتصفح وتأكد أنك ترى المنتج بسعره

السبب 2: المتجر غير مدعوم
  → جرّب متجراً من المدعومين (أمازون، شوبيفاي، ووكومرس)
  → الحل: استخدم "🏬 طلب إضافة متجر"

السبب 3: المتجر يحظر البوت
  → بعض المواقع تكشف السكرابينج وتحظره
  → الحل: انتظر وجرّب مرة أخرى بعد بضع دقائق

السبب 4: الرابط محمي بـ SSRF
  → إذا كان الرابط يؤدي لعنوان داخلي أو خاص (127.0.0.1, 192.168.x.x...)
  → النظام يرفضه لأسباب أمنية. هذا سلوك طبيعي.

السبب 5: مشكلة مؤقتة في الموقع
  → الموقع قد يكون بطيئاً أو معطلاً
  → الحل: جرّب مرة أخرى لاحقاً


──────────────────────────────────────
❌ "وصلت للحد الأقصى!"
──────────────────────────────────────

لماذا يحدث:
  كل خطة لها حد لعدد المنتجات: مجاني = 3، أساسي = 50، احترافي = 300

كيف تتحقق من عدد منتجاتك:
  اضغط "📦 منتجاتي" → العدد يظهر في العنوان: "(3 منتج)"

كيف تحل المشكلة:
  الخيار 1: احذف منتجات لم تعد تحتاجها
    → من "📦 منتجاتي" → اضغط على منتج → "🗑 حذف المنتج"
  الخيار 2: رقّ اشتراكك
    → من "💳 الاشتراك" → "⬆️ ترقية الآن"


──────────────────────────────────────
❌ البوت لا يرسل تنبيهات رغم انخفاض السعر
──────────────────────────────────────

قائمة فحص:

  □ هل نوع التنبيه مفعّل؟
    → من "📦 منتجاتي" → اضغط المنتج → "🔔 إعداد التنبيهات"
    → تأكد أن "📉 انخفاض السعر" أو التنبيه المطلوب يحمل علامة ✅

  □ هل خاصية "كتم التنبيهات" مفعّلة؟
    → من "⚙️ الإعدادات" → تحقق من: "التنبيهات: مفعلة ✅"
    → إذا "مكتومة ⏸" → اضغط "🔕 كتم التنبيهات" لإلغاء الكتم

  □ هل المراقبة موقوفة مؤقتاً؟
    → من التفاصيل: إذا الحالة "⏸" → اضغط "▶️ استئناف المراقبة"

  □ هل الخصم كافٍ للتنبيه نوع "خصم كبير فقط"؟
    → يحتاج 20% على الأقل

  □ هل المنتج تم فحصه مؤخراً؟
    → من تفاصيل المنتج: تحقق "⏰ آخر تحديث"
    → إذا قديم جداً → النظام قد يكون متوقفاً


──────────────────────────────────────
❌ لوحة الإدارة لا تفتح (This site can't be reached)
──────────────────────────────────────

السبب 1: البوت (main.py) ليس قيد التشغيل
  → تأكد من تشغيل: python main.py
  → ابحث في الطرفية عن: "✅ لوحة الإدارة: http://0.0.0.0:8000"

السبب 2: المنفذ 8000 مشغول
  → في الطرفية: netstat -ano | findstr :8000
  → إذا وجدت عملية → أوقفها أو غيّر المنفذ في .env:
    DASHBOARD_PORT=8001

السبب 3: جدار الحماية يحظر المنفذ
  → إذا تحاول الوصول من جهاز آخر → افتح المنفذ 8000 في الجدار

السبب 4: لا يمكنك تسجيل الدخول
  → تأكد أن Telegram ID الخاص بك موجود في ADMIN_USER_IDS في .env


──────────────────────────────────────
❌ السكرابينج يفشل دائماً لمتجر معين
──────────────────────────────────────

الأسباب المحتملة:
  1. المتجر يستخدم JavaScript كثيف (مثل React SPA) → الموصل العام لا يقرأ JS
  2. المتجر يحظر عناوين IP الخوادم
  3. هيكل HTML تغير (خاصة أمازون تغير باستمرار)

الحلول:
  1. استخدم "🏬 طلب إضافة متجر" لإعلام الإدارة
  2. الإدارة يمكنها إنشاء connector مخصص لهذا المتجر
  3. لأمازون: تحقق من CSS selectors في core/connectors/amazon.py

════════════════════════════════════════
الجزء 10: نظام الدعم الفني
════════════════════════════════════════

نظرة عامة:
──────────

نظام الدعم الفني يربط بين 3 واجهات في نفس الوقت:

1. المستخدم داخل البوت:
   - يفتح تذكرة من زر "🎧 الدعم الفني".
   - يختار القسم المناسب: خدمة العملاء، الفواتير، أو الدعم التقني.
   - يرسل أول رسالة، فيُنشئ النظام تذكرة جديدة برقم مثل: TKT-2026-0001.
   - بعد ذلك، أي رد جديد في وضع التذكرة يُضاف لنفس المحادثة.

2. فريق الإدارة على تيليغرام:
   - تصله رسالة فورية في مجموعة الإدارة عند وصول تذكرة جديدة.
   - تحتوي الرسالة على رقم التذكرة، اسم المستخدم، القسم، ونص أول رسالة.
   - يوجد زر "⚡ رد سريع" لبدء الرد من تيليغرام مباشرة بدون فتح لوحة التحكم.

3. لوحة الإدارة:
   - تعرض قائمة التذاكر على اليسار، والمحادثة الكاملة على اليمين.
   - يمكن الرد، النقل بين الأقسام، التعيين لأحد أعضاء الفريق، الحل، أو الإغلاق.
   - التحديثات تصل مباشرة عبر WebSocket، ومع فشل الاتصال ينتقل النظام إلى مزامنة دورية.


كيف يفتح المستخدم تذكرة؟
─────────────────────────

1. يضغط المستخدم على: 🎧 الدعم الفني.
2. تظهر له قائمة فيها:
   - فتح تذكرة جديدة
   - آخر التذاكر المفتوحة (إن وجدت)
   - رجوع للرئيسية
3. يختار القسم المناسب.
4. يكتب رسالته الأولى.
5. النظام ينشئ:
   - SupportTicket
   - SupportMessage أولى مرتبطة بالتذكرة
   - إشعار إدارة في لوحة التحكم
   - إشعار تيليغرام لمجموعة الإدارة
6. يحصل المستخدم على رسالة تأكيد تحتوي على رقم التذكرة.


كيف يرد المدير؟
────────────────

هناك طريقتان للرد:

أولاً: من لوحة الإدارة
  - افتح: /support
  - اختر التذكرة من القائمة اليسرى
  - اكتب الرد في مربع المحادثة
  - اضغط "إرسال"
  - النتيجة:
    • يُحفظ الرد في SupportMessage
    • تتغير حالة التذكرة إلى waiting_user
    • يُرسل الرد للمستخدم في تيليغرام فوراً

ثانياً: من مجموعة الإدارة في تيليغرام
  - اضغط على زر "⚡ رد سريع"
  - أرسل الرسالة النصية
  - البوت يربط الرسالة بالتذكرة ويعيد إرسالها للمستخدم


إدارة الفريق:
─────────────

من صفحة /support/team يمكن إدارة أعضاء الفريق المسؤولين عن التذاكر.

لكل عضو فريق توجد البيانات التالية:
  • الاسم المعروض
  • القسم
  • الارتباط بحساب admin_users (اختياري لكن مهم للإشعارات)
  • اللون المستخدم للصورة الرمزية
  • حالة التوفر
  • إحصائيات: عدد التذاكر ومتوسط زمن الرد

عند تعيين تذكرة لعضو فريق:
  • يتم تحديث assigned_to في التذكرة
  • تُضاف رسالة نظام داخل المحادثة
  • يمكن إرسال إشعار تيليغرام للعضو إذا كان لديه telegram_id مرتبط


حالات التذكرة:
──────────────

  open          → تذكرة جديدة لم يبدأ التعامل معها
  in_progress   → الفريق يعمل عليها حالياً
  waiting_user  → تم الرد من الإدارة والكرة الآن عند المستخدم
  resolved      → تم حل المشكلة
  closed        → أُغلقت التذكرة نهائياً


الجداول المرتبطة بالدعم:
───────────────────────

  support_tickets   → بيانات التذكرة الرئيسية
  support_messages  → جميع الرسائل داخل المحادثة
  team_members      → أعضاء فريق الدعم

الحقول الزمنية في نظام الدعم تُخزن بصيغة timezone-aware UTC.
لوحة الإدارة تتولى تحويلها محلياً في الواجهة حتى لا يظهر فرق زمني خاطئ.


التحديثات اللحظية:
──────────────────

القنوات المستخدمة:
  admin:notifications  → إشعارات الجرس في لوحة الإدارة
  support:messages     → تحديثات الرسائل والتذاكر

الواجهات المستخدمة:
  GET  /api/support/tickets
  GET  /api/support/tickets/{ticket_id}
  POST /api/support/tickets/{ticket_id}/reply
  POST /api/support/tickets/{ticket_id}/assign
  POST /api/support/tickets/{ticket_id}/transfer
  POST /api/support/tickets/{ticket_id}/resolve
  POST /api/support/tickets/{ticket_id}/close
  GET  /api/support/team
  POST /api/support/team
  PUT  /api/support/team/{member_id}
  GET  /api/support/stats
  WS   /ws/support/{ticket_id}


ملاحظات تشغيل مهمة:
───────────────────

1. إذا لم تُشغل Migration الخاصة بالدعم، فإن صفحات الدعم لن تجد الجداول المطلوبة.
2. إذا كان Redis غير متاح، سيبقى الحفظ في قاعدة البيانات يعمل لكن التحديثات الفورية ستضعف.
3. إذا انقطع WebSocket في لوحة الإدارة، تتحول الصفحة إلى مزامنة دورية تلقائياً.

الجزء 11: صفحة الإعدادات (Dashboard)
════════════════════════════════════════

صفحة الإعدادات متاحة في لوحة الإدارة على المسار:
  /settings

الصفحة تحتوي 8 تبويبات مستقلة، وكل تبويب له زر حفظ خاص به:

1. 🤖 البوت
   - اسم البوت
   - رسالة /start
   - وضع الصيانة ورسالة الصيانة
   - الحد الأدنى للخصم
   - وضع الاختبار

2. 💳 الاشتراكات
   - أسعار الخطط
   - حدود المنتجات/الفئات/المتاجر
   - فترات الفحص لكل خطة
   - عرض وإدارة كودات الخصم

3. ⏱ المراقبة
   - التأخير بين الطلبات
   - حد الطلبات لكل دقيقة
   - أقصى منتجات في الدورة
   - محاولات إعادة المحاولة
   - عرض حالة المواقع

4. 🔔 الإشعارات
   - قوالب رسائل التنبيهات القابلة للتعديل
   - متغيرات مثل {product_name} و {discount}

5. 👥 الفريق
   - إدارة أعضاء فريق الدعم
   - تحديث بيانات العضو وتوفّره
   - إنشاء عضو جديد
   - عرض سجل تسجيل الدخول (واجهة متابعة)

6. 💰 الأفلييت
   - الرابط الافتراضي
   - الوسم التلقائي (Tag)
   - تفعيل/تعطيل المنصات

7. 🔒 الأمان
   - محاولات تسجيل الدخول
   - مدة القفل
   - مدة JWT
   - Rate Limit
   - قائمة IPs المحظورة

8. 🛠 النظام
   - معلومات إصدار النظام
   - حالة Redis
   - آخر Migration
   - تشغيل أدوات الصيانة والتصدير


كيف تُحفظ الإعدادات؟
───────────────────

يتم حفظ جميع الإعدادات في جدول: bot_settings

كل سطر يتكون من:
  key        مثل: bot.welcome_message
  value      قيمة نصية
  value_type نوع القيمة (string/integer/float/boolean/json)
  category   تصنيف (bot/plans/monitoring/templates/affiliate/security)

الواجهة المستخدمة:
  GET  /api/settings/{category}
  POST /api/settings/{category}

عند الحفظ:
  • تُحفظ القيمة في قاعدة البيانات
  • تُحدث بعض القيم في الذاكرة مباشرة (مثل PLAN_LIMITS)
  • يُسجل الحدث في AuditLog


واجهات النظام المساندة:
──────────────────────

  GET  /api/settings/system/info
  POST /api/settings/system/restart-monitor
  POST /api/settings/system/clear-cache
  GET  /api/settings/system/export/{type}

الأنواع المدعومة للتصدير:
  users | products | reports

الجزء 13: منشئ قائمة البوت (Bot Menu Builder)
════════════════════════════════════════

ما هو منشئ قائمة البوت؟
──────────────────────

منشئ قائمة البوت هو أداة إدارة قوية في لوحة التحكم تسمح للمسؤولين بـ:

  • إنشاء أزرار جديدة للبوت بدون الحاجة لكتابة أي أكواد برمجية
  • تعديل الأزرار الموجودة (النص، الرمز التعبيري، الإجراء)
  • حذف الأزرار القديمة أو غير المستخدمة
  • إعادة ترتيب الأزرار بسهولة (سحب وإفلات)
  • التحكم في ظهور الأزرار حسب خطة الاشتراك للمستخدمين
  • اختيار نوع الإجراء: وسط (menu)، رسالة (message)، رابط (url)، أمر (command)، دالة معالج (handler)

كل هذا يحدث فوراً بدون الحاجة لإعادة تشغيل البوت.


الوصول إلى منشئ قائمة البوت:
──────────────────────────

من لوحة التحكم (Dashboard):
  1. انتقل إلى: http://localhost:8000 (أو عنوان الخادم الخاص بك)
  2. ابحث عن قائمة الملاحة الجانبية على اليسار
  3. اضغط على الخيار: "🎛 منشئ القائمة" (Menu Builder)
  4. ستُفتح الواجهة البصرية الكاملة لإدارة الأزرار


واجهة المستخدم — 3 أقسام رئيسية:
──────────────────────────────

┌─────────────────────────────────────────────────────────────────┐
│                    منشئ قائمة البوت                            │
├────────────────┬────────────────────────────┬────────────────────┤
│                │                            │                    │
│   (يسار)       │        (وسط)              │      (يمين)        │
│                │                            │                    │
│  معاينة الهاتف  │  منطقة البناء والسحب       │  لوحة التعديل      │
│                │  (Drag & Drop)             │                    │
│  تظهر كيف       │                            │  تعديل خصائص       │
│  الأزرار        │  اسحب الأزرار لترتيبها     │  الزر المختار:     │
│  ستظهر للمستخدم │  من اليمين إلى اليسار     │  • العنوان         │
│                │                            │  • الرمز           │
│                │  ← اسحب الأزرار هنا        │  • نوع الإجراء     │
│                │                            │  • قيمة الإجراء    │
│                │  اضغط + لإضافة زر جديد     │  • الموضع          │
│                │  اضغط × لحذف زر           │  • الظهور          │
│                │                            │                    │
└────────────────┴────────────────────────────┴────────────────────┘

القسم الأيسر — معاينة الهاتف (Phone Preview):
───────────────────────────────────
  • يعرض الأزرار بنفس الطريقة التي ستظهر بها للمستخدمين
  • معاينة حية: أي تعديل تفعله يظهر فوراً
  • يتعامل مع النصوص الطويلة والرموز بشكل صحيح
  • يحترم اتجاه النصوص (RTL للعربية)


القسم الأوسط — منطقة البناء (Builder Panel):
──────────────────────────────────
  • تحتوي على قائمة جميع الأزرار الموجودة
  • كل زر يمكن سحبه لإعادة ترتيبه
  • يظهر بجانب كل زر:
    - 🟩 مربع ملوّن للسحب (drag handle)
    - اسم الزر
    - حالته (🔴 غير مفعّل / 🟢 مفعّل)
  
  أزرار التحكم:
    ➕ إضافة زر جديد:
       - ينشئ زر افتراضي جديد
       - المحرر يفتح تلقائياً على اليمين
       - اكتب الاسم وحدد الإجراء ثم احفظ
    
    × حذف زر:
       - يظهر زر حذف بجانب كل زر
       - ستُطلب منك تأكيد قبل الحذف
       - الحذف نهائي ولا يمكن الرجوع
    
    💾 حفظ التغييرات:
       - يُحفظ الترتيب الجديد تلقائياً عند إعادة الترتيب
       - التعديلات على الخصائص تُحفظ عند الضغط على "حفظ"


القسم الأيمن — لوحة التعديل (Edit Panel):
────────────────────────────────────
  عندما تختار زراً أو تنشئ زراً جديداً، ستظهر هذه الحقول:

  1️⃣ العنوان (Label):
     - النص الذي سيظهر على الزر
     - مثال: "➕ إضافة منتج"، "📦 منتجاتي"
     - يمكن أن يحتوي على رموز تعبيرية (emojis)

  2️⃣ الرمز التعبيري (Emoji):
     - ختيار سريع من 32 رمزاً شائعاً
     - اضغط على زر "🌈" لفتح منتقي الرموز
     - أو اطبع الرمز مباشرة بجانب العنوان

  3️⃣ نوع الإجراء (Action Type) - القائمة المنسدلة:
     
     • menu (الافتراضي): قائمة فرعية
       → عند الضغط، يُظهر أزراراً فرعية إضافية
     
     • message: رسالة نصية
       → عند الضغط، يرسل رسالة نصية للمستخدم
       → يمكن أن تكون شرح أو إعلان
     
     • url: رابط خارجي
       → عند الضغط، يفتح رابط (أفلييت، متجر، إلخ)
       → R̄abط يجب أن يبدأ بـ http:// أو https://
     
     • command: أمر بوت
       → عند الضغط، يُشغّل أمر (/start، /help، إلخ)
     
     • handler: معالج مخصص
       → عند الضغط، ينفذ دالة معالج محددة مسبقاً
       → أمثلة: "add_product"، "my_products"، "best_deals"
     
     • support: نظام الدعم الفني
       → عند الضغط، يفتح قائمة الدعم
     
     • subscribe: نظام الاشتراك/الترقية
       → عند الضغط، يعرض خيارات الاشتراك

  4️⃣ قيمة الإجراء (Action Value) - يختلف حسب النوع:
     
     • إذا كان message: النص الذي سيُرسل
     • إذا كان url: الرابط الكامل
     • إذا كان command: اسم الأمر (بدون /)
     • إذا كان handler: اسم الدالة (add_product, my_products, etc.)
     • إذا كان menu أو support أو subscribe: فارغ (لا يحتاج قيمة)

  5️⃣ الموضع في القائمة (Position):
     • الصف (Row): أي صف من أسفل الشاشة؟
     • العمود (Column): أي موقع يميناً؟
     • يُتحسب تلقائياً عند السحب، لكن يمكن تعديله يدوياً

  6️⃣ الظهور حسب نوع الخطة (Visibility):
     • all: يظهر لجميع المستخدمين
     • free: فقط مستخدمو الخطة المجانية
     • basic: خطة أساسية فما فوق
     • professional: خطة احترافية فما فوق
     • admin: فقط للمسؤولين


خطوات عملية — إنشاء زر جديد:
──────────────────────────

مثال: إضافة زر "🆕 ميزة جديدة"

  Step 1: في القسم الأوسط، اضغط على ➕ "إضافة زر جديد"
  Step 2: سيظهر زر جديد بعنوان مؤقت، و"لوحة التعديل" تفتح على اليمين
  Step 3: في حقل "العنوان"، اكتب: "🆕 ميزة جديدة"
  Step 4: اختر النوع من القائمة المنسدلة (مثل "handler")
  Step 5: أدخل قيمة الإجراء (مثل "new_feature_handler")
  Step 6: حدد الظهور (مثل "professional" للعملاء المميزين فقط)
  Step 7: اضغط زر 💾 "حفظ"
  Step 8: سيظهر الزر الجديد في لوحة المعاينة على الفور!


خطوات عملية — تعديل زر موجود:
───────────────────────────

مثال: تغيير الزر "📦 منتجاتي" ليقول "📦 مشترياتي"

  Step 1: اضغط على الزر في القسم الأوسط (منطقة البناء)
  Step 2: ستظهر تفاصيله في "لوحة التعديل" على اليمين
  Step 3: غيّر النص من "منتجاتي" إلى "مشترياتي"
  Step 4: اضغط 💾 "حفظ"
  Step 5: الزر محدّث فوراً!


خطوات عملية — حذف زر:
───────────────────

مثال: حذف الزر "🏬 طلب متجر جديد"

  Step 1: اضغط على الزر في القسم الأوسط
  Step 2: اضغط زر 🗑 "حذف"
  Step 3: تأكيد: "هل أنت متأكد من رغبتك في حذف هذا الزر؟"
  Step 4: اضغط OK للتأكيد
  Step 5: الزر حُذف من القائمة والمعاينة


خطوات عملية — إعادة ترتيب الأزرار (سحب وإفلات):
────────────────────────────────────

  Step 1: في القسم الأوسط، لاحظ المربع الملوّن 🟩 بجانب كل زر
  Step 2: اسحب زراً من المربع الملوّن هذا
  Step 3: اسحبه لأعلى أو أسفل في القائمة
  Step 4: أفلته (اترك الماوس)
  Step 5: القائمة ستُعاد ترتيبها فوراً!
  Step 6: المعاينة على اليسار ستُحدّث تلقائياً


خطوات عملية — نشر التغييرات على البوت:
───────────────────────────────────

بعد كل إضافة أو تعديل أو حذف:

  Method 1 — تلقائي:
    • التغييرات تُحفظ في قاعدة البيانات فوراً
    • البوت يتحقق من التحديثات كل 60 ثانية
    • الأزرار الجديدة ستظهر تلقائياً

  Method 2 — يدوي (فوري):
    • في أعلى لوحة المعاينة، ابحث عن زر 🚀 "نشر الآن"
    • اضغطه لإرسال الأمر للبوت فوراً
    • البوت سيحدّث قائمته في ثانية واحدة


تلميحات واختصارات مفيدة:
─────────────────────

  ⌨️ لوحة المفاتيح:
    • Tab: الانتقال بين الحقول
    • Enter: حفظ بسرعة (في بعض الحقول)

  🎨 الرموز التعبيرية:
    • استخدم رموز معبّرة لتسهيل التعريف بالأزرار
    • مثالاً: ➕, 📦, 🔥, 💳, ❓, ⚙️, إلخ
    • تجنب الرموز الكبيرة جداً (قد لا تظهر بشكل صحيح)

  🔄 سهولة التبديل:
    • يمكنك التعديل على عدة أزرار الواحد تلو الآخر
    • لا تقلق! التغييرات تُحفظ تلقائياً

  🧪 الاختبار:
    • بعد نشر الأزرار، افتح البوت واختبرها
    • اضغط على الزر الجديد للتأكد من أنه يعمل
    • إذا حدث خطأ، عُد لمحرر القائمة وعدّل


معالجات الأخطاء الشائعة:
───────────────────

✗ "الزر الجديد لا يظهر في البوت"
  ✓ تأكد من أن الزر "مفعّل" (🟢 أخضر في المعاينة)
  ✓ تحقق من قيمة الظهور (Visibility) — هل تتطابق مع خطة المستخدم؟
  ✓ انتظر 60 ثانية أو اضغط 🚀 "نشر الآن"

✗ "رسالة الخطأ: لا يمكن حفظ الزر"
  ✓ تأكد من ملء جميع الحقول المطلوبة
  ✓ إذا كان نوع الإجراء "url"، تأكد من الرابط يبدأ بـ http(s)://
  ✓ تحقق من اتصالك بالإنترنت

✗ "الزر يعمل لكن النص غير صحيح"
  ✓ عد للمحرر وصحح النص
  ✓ احفظ التغييرات
  ✓ اختبر مرة أخرى

✗ "الزر يختفي بعد إعادة تشغيل البوت"
  ✓ قد تكون الخطة غير صحيحة (تحقق من Visibility)
  ✓ أو قد كانت هناك مشكلة في الحفظ
  ✓ تأكد بعد الإضافة أن الزر موجود في البائمة


البيانات المشفرة والآمنة:
──────────────────────

✅ كل تعديل على القائمة يُحفظ مباشرة في قاعدة البيانات
✅ المسؤول الوحيد الذي يمكنه تغيير الأزرار
✅ كل تعديل يُسجّل في الأرشيف (audit log)
✅ لا توجد معلومات حساسة تُعرّض في المحرر
